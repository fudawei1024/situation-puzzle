# 《海龟汤·境》交互革新版 - 详细设计文档

## 1. 产品概述与设计哲学

### 1.1 产品定位
一款**无需打字、无需语音**的情境推理解谜游戏。玩家通过组合“侦探关键词卡牌”来向AI主持人提问，逐步还原离奇事件的真相。它保留了海龟汤“无限提问、顿悟真相”的内核，但将交互重塑为符合移动端碎片化场景的卡牌策略游戏。

### 1.2 核心设计原则
- **零输入门槛**：所有提问通过点选、拖拽卡牌完成，老年人与儿童皆可上手。
- **思考视觉化**：线索以便签、连线、关系网形式自动呈现，推理过程一目了然。
- **引导优于自由**：用卡牌框架引导新手玩家“如何提问”，同时为高手保留自由输入出口。
- **操作即乐趣**：卡牌的组合、解锁、消耗本身就是一种轻策略玩法，而不仅是工具。

## 2. 核心玩法：关键词卡牌系统

### 2.1 卡牌定义
每一张卡牌代表一个**推理要素**或**逻辑动作**。玩家通过组合卡牌来构造问题。

#### 2.1.1 卡牌分类
| 大类 | 子类 | 卡牌示例 | 颜色/图标 |
| :--- | :--- | :--- | :--- |
| **要素卡** (Who/What/Where/When) | 人物 | 死者、发现者、嫌疑人、目击者、朋友、家人 | 深红色、人形图标 |
| | 地点 | 案发现场、附近、关联地点、远方 | 墨绿色、图钉图标 |
| | 物品 | 凶器、遗留物、信件、电话、衣物 | 古铜色、钥匙图标 |
| | 时间 | 案发前、案发时、案发后、具体时刻 | 暗蓝色、钟表图标 |
| **动作卡** (Relation/Action/Motive) | 关系 | 认识？、亲密？、敌对？、合作？ | 紫色、连线图标 |
| | 行为 | 做了什么？、计划？、掩盖？ | 橙色、齿轮图标 |
| | 动机 | 金钱、情感、复仇、意外、保护 | 血色、心碎图标 |
| | 状态 | 死亡原因、死亡方式、心理状态、谎言 | 灰色、放大镜图标 |

#### 2.1.2 卡牌属性
- **稀有度**：普通（白）、稀有（蓝）、史诗（紫）。稀有卡牌指向更关键或易被忽略的线索，由系统根据谜题动态投放。
- **关联词**：每张卡牌预置了多个自然语言模板，用于生成问题（详见2.3）。

### 2.2 手牌管理
玩家不是拥有全部卡牌，而是像卡牌游戏一样，拥有一个动态的手牌区。

- **初始手牌**：每局开始，系统根据汤面自动发放6-8张基础卡牌（包含通用要素和少量动作卡）。这些卡牌保证玩家能开始提问。
- **卡牌获取（探索奖励）**：
  - 当主持人回答“是”时，有概率解锁1-2张与该线索相关的新卡牌，加入手牌。例如，问到“有目击者吗？”回答“是”，则手牌中可能解锁“目击者”这张新人物卡。
  - 使用“灵感一闪”提示时，必定获得一张稀有卡牌。
- **手牌上限**：通常保持8-12张。超过上限时，玩家需选择弃掉一张或自动替换最旧卡牌。
- **卡牌消耗**：多数提问不会消耗卡牌。但部分强力“动作卡”使用后可能进入冷却时间或被弃掉，增加策略性。

### 2.3 问题构造逻辑
玩家通过**选择1张要素卡 + 1张动作卡**来构造问题。系统实时预览完整问句，确认后发送。

#### 2.3.1 组合规则表（示例）
| 要素卡 | 动作卡 | 生成问句示例 |
| :--- | :--- | :--- |
| 死者 | 职业 | “死者从事特殊职业吗？” |
| 嫌疑人 | 关系(与死者) | “嫌疑人和死者认识吗？” |
| 凶器 | 状态 | “凶器是提前准备好的吗？” |
| 案发现场 | 时间(案发时) | “案发时现场还有别人吗？” |
| 朋友 | 动机(情感) | “朋友的动机与感情有关吗？” |
| 遗留物 | 行为(掩盖) | “遗留物是凶手故意留下的吗？” |

- **单一卡牌提问**：也可以只选一张要素卡并点击“详情”按钮，系统生成该要素的常规问题，如“现场有打斗痕迹吗？”“死者死因是中毒吗？”
- **智能预览**：在底部问题构造区，选中卡牌后，立即显示完整问句，玩家可滑动切换该卡牌的备选问句（每对组合可能有多达3种问法），选中最贴合自己想法的发送。

#### 2.3.2 特殊卡牌
- **“为什么”万能卡**：每局限用3次。直接对任一已有线索提问“为什么？”，主持人会尝试更详细的叙述，但可能仍回答“不重要”。
- **“验证猜想”卡**：玩家在推理板上形成逻辑链后，可对某个“假想结论”使用此卡，系统生成“真相是……吗？”的问句。

### 2.4 卡牌界面与交互
#### 2.4.1 卡牌展示
- 每张卡牌为圆角矩形卡片，尺寸约70x100px（小游戏屏幕适配），正面包含图标、名称、稀有度光效。
- 手牌区位于屏幕底部，采用横向滑动选择，当前选中的卡片会轻微放大并上浮。
- 卡牌有精细的微动效：待选时轻微呼吸光效，选中时边缘亮起金边，拖拽时跟随手指带投影。

#### 2.4.2 操作流程
1.  **选择第一张要素卡**：在底部手牌区左滑/右滑浏览，点击一张要素卡（如“死者”）。它跳入屏幕中下方的“问题槽”左侧。
2.  **选择第二张动作卡**：手牌区自动切换为与该要素卡兼容的动作卡列表（过滤不相关动作）。玩家点击动作卡（如“死亡方式”），它跳入问题槽右侧。
3.  **预览与确认**：问题槽中央显示生成的完整问句。玩家可上下滑动切换备选问句。
4.  **发送**：点击问题槽右侧的发送按钮（或再次点击槽位），问句以聊天气泡形式发出，主持人开始回应。
5.  **取消**：点击槽外区域或按取消按钮，清空选择。

- **快速提问**：双击某张要素卡，直接用默认动作卡（“详情”）生成问题并发送，省略第二选择步骤，适合快节奏。
- **拖拽组合**：高级玩家可将两张卡牌直接拖拽到一起，触发快速组合。

### 2.5 卡牌与AI主持人的配合
- 系统接收的是“卡牌组合ID”，而非自然语言。这完全规避了语音识别和NLU的不可靠性。
- 主持人根据预设规则库（谜题配置的关键线索列表）判定答案。例如，若“死者+死亡方式”对应谜底中的“被毒杀”，则回答“是”，并可能触发获得新卡牌“毒药”。
- 若组合不在预设线索中，且不属于强干扰项，则回答“不重要”。系统会记录此组合，用于避免重复无用提问。
- 主持人的性格化反馈仍保留，根据答案类型和上下文播放不同语音/动画。

## 3. 侦探推理板系统

### 3.1 自动记录与视觉呈现
- 当玩家提问获得“是”的回答时，该问题中的关键要素（如“死者认识凶手”）自动生成一张**线索便签**。
- 便签自动钉在推理板上，初始随机倾斜角度，模拟软木板质感。
- 便签内容为简短线索文本，下方标注来源（卡牌组合或自由提问）。

### 3.2 推理板交互
- **进入**：点击主界面右上角图钉图标，展开全屏推理板。
- **整理线索**：
  - **拖拽**：手指可拖动任意便签到任意位置。
  - **分组**：将两张便签拖到一起，自动形成虚线框将它们归为一组，可命名组标题（如“动机相关”）。
  - **连线**：从一张便签向另一张滑动手指，生成一条红色棉线连接两者，表示逻辑关联。点击线可删除。
  - **高亮**：双击某便签，它变为高亮状态，方便讲解或截图。
- **新建假说**：双击空白处，创建一张颜色不同的“假说便签”，玩家可输入简短推论（通过语音或手写，但可预设为纯手写/选择模板词）。这些便签可用于提交最终推理。
- **自动整理**：提供“智能布局”按钮，根据时间流、人物关系或因果链自动排列便签。这是AI驱动的小功能，提升体验。

### 3.3 推理板与卡牌联动
- 在推理板中，长按一张线索便签，弹出选项“追问”，系统自动将该线索对应的要素卡与相关动作卡调入问题槽，方便玩家继续深挖。
- 推理板中可随时将“假说便签”拖到“提交推理”区域（底部中心），作为最终答案的一部分。

## 4. 替代与增强交互模式

### 4.1 沉浸式场景模式（高级谜题专用）
- 适用于剧情战役或高品质DLC谜题。汤面展示为一幅精美插画（犯罪现场、书房等）。
- 点击画面中可疑物品（如倒下的椅子、打开的窗户），直接生成对应问题。实质是将“要素卡”视觉化为场景物品。
- 右上角保留卡牌栏入口，玩家可切换回卡牌模式。

### 4.2 自由输入模式（保留给核心玩家）
- 在卡牌栏最右侧固定一个“键盘”图标。点击后展开传统输入框和语音按钮。
- 自由输入的问题同样会被解析，若匹配到预设线索，同样触发线索便签和卡牌解锁。
- 自由模式下，系统会记录玩家提问文本，用于优化卡牌系统未覆盖的提问路径。

## 5. 用户界面详细设计

### 5.1 主游戏界面（竖屏）
```
┌──────────────────────────┐
│  🔙 谜题名       📌推理板│  (顶栏，高44px)
├──────────────────────────┤
│                          │
│     🎩 主持人形象区      │  (占屏高25%，动态立绘+表情)
│    (半身像，背景氛围)    │
│                          │
├──────────────────────────┤
│                          │
│   💬 对话历史区          │  (占屏高40%，带聊天气泡)
│   玩家问...              │
│   主持人：是。           │
│   ...                    │
│                          │
├──────────────────────────┤
│  [问题预览区]  发送→    │  (高40px，显示当前组合问句)
├──────────────────────────┤
│  [🃏卡牌栏 - 可滑动]    │  (高100px，3行或大卡横滑)
│  [死者][嫌疑人][动机]... │
└──────────────────────────┘
```

- **问题预览区**：常驻显示即将发送的问题。选中卡牌后，此处文字动态更新。右端为发送按钮（纸飞机图标），左端有撤销按钮。
- **卡牌栏**：
  - 主区域显示手牌，每张卡牌为竖版卡牌样式。
  - 卡牌栏上方有分类标签（全部/人物/物品/动机/行为），点击筛选。
  - 最左侧有“🎤自定义”按钮，点击切换至自由输入。
  - 卡牌稀有度通过边框颜色和轻微光效体现。
- **对话区**：聊天气泡，玩家问题居右（深色底），主持人回答居左（浅色底）。“是”气泡左侧有金色标记，“否”为灰色，“不重要”为半透明。
- **主持人区**：支持轻微动画（点头、摇头、摊手），背景随谜题氛围变化（书房、雾港、医院等）。

### 5.2 推理板界面
- 全屏深色软木板纹理背景。
- 便签为微黄方形纸片，带图钉图形，文字使用手写体。
- 底部工具栏：连线模式、新建假说便签、自动布局、垃圾桶（删除）、关闭。
- 顶部显示当前谜题名和“提交推理”按钮。

### 5.3 揭晓仪式界面
- 提交推理后，画面渐变至全黑，然后出现旧电影倒计时效果。
- 汤底文字以打字机效果逐行出现，关键句放大并伴有重音音效。
- 背景出现与该谜题匹配的微动画（雨、烛火、飘落的信纸等）。
- 最后显示侦探评级和一张精美的“结案报告”卡片，包含生成的小程序分享码。

## 6. 动态难度与引导系统

### 6.1 初始手牌难度调节
- 简单谜题：初始手牌包含更直接的要素（如“凶器”、“死亡方式”），稀有卡初始可见。
- 困难谜题：初始手牌更泛化（如“事件”、“人物”），关键卡牌需自己通过提问解锁。

### 6.2 智能引导（无感提示）
- 若玩家连续3次获得“不重要”，主持人在回答后会自动追加一句引导语，同时卡牌栏中一张更可能的卡牌会短暂高亮闪烁。
- 若玩家重复提问相似组合，主持人会提醒“这个问题我们已经确认过了，不妨换个角度”，并将已使用的卡牌置灰片刻。

### 6.3 提示系统（激励视频入口）
- **卡牌提示**：看广告后，直接在手牌中添加一张高价值稀有卡。
- **组合提示**：看广告后，问题预览区自动填入一个高效问题组合，玩家可直接发送。
- **线索揭示**：看广告后，推理板上自动生成一条关键线索便签。

## 7. 谜题数据结构扩展

为了支持卡牌系统，每个谜题配置需增加字段：

| 字段 | 描述 | 示例 |
| :--- | :--- | :--- |
| 汤面 | 短文本 | “男子走进酒吧要了杯水，酒保却拿枪指着他……” |
| 汤底 | 完整故事 | “男子打嗝，要水止嗝，酒保误以为抢劫……” |
| 核心谜底 | 1句话 | “男子打嗝，酒保误解了” |
| **卡牌答案映射表** | 核心数据！定义哪些卡牌组合会得到“是”。 | `{"死者+死亡方式": "否", "死者+行为": "是", ...}` |
| **初始手牌配置** | 该谜题开局发放的卡牌ID列表 | `["死者","地点","动机","行为","关系"]` |
| **关键解锁牌** | 回答“是”时概率解锁的卡牌及其条件 | `{"线索_打嗝": ["症状卡","疾病卡"]}` |
| 干扰映射表 | 某些组合会得到“否”或“不重要” | 用于阻止卡牌组合产生矛盾 |
| 氛围背景 | 主持人背景图ID | `bg_bar` |
| 揭晓BGM | 音乐资源ID | `music_reveal_01` |

卡牌本身是全局的，由系统提供约200-300张通用卡牌。但每张卡牌在不同谜题中可能映射不同的答案，这部分完全由“卡牌答案映射表”控制。

## 8. 社交裂变设计（适配卡牌交互）

- **求助分享**：卡关时，可分享当前推理板截图和一则“我需要灵感”的消息到微信群。朋友点击后，可赠送一张稀有卡牌或直接发一条提示消息给玩家。
- **好友推理对战**：双方使用相同初始手牌和谜题，比拼最少提问次数和评级。对战结束后，可互相查看对方的推理板布局，增加策略交流。
- **卡牌交易所**：在玩家社区中，可以互相赠送或交换稀有卡牌（仅限非谜题限定的通用卡牌），用于装饰或收藏，不与谜题解法挂钩，避免作弊。
- **每日一谜全球挑战**：所有玩家使用相同初始卡牌配置，在排行榜上比拼S级达成率。

## 9. 技术实现要点

- **卡牌系统客户端**：使用数据驱动，所有卡牌组合的文本生成由本地规则引擎完成，无需网络请求。答案判定优先本地映射表，复杂谜题可异步服务端校验。
- **卡牌资源分包**：所有卡牌图标、背景、音效作为资源包，首包极小。
- **AI辅助**：自由输入模式下的语义匹配、智能引导卡牌推荐、推理板自动布局等，可调用云端轻量NLP服务，按需加载。
- **数据埋点**：详尽记录每局卡牌使用频次、组合成功率、卡关点，用于优化谜题配置和卡牌设计。
